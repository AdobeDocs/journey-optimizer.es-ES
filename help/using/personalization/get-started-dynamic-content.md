# Introducción al contenido dinámico {#start-dynamic-content}

>[!BEGINSHADEBOX]

**En esta página:** Comprenda cómo el contenido dinámico utiliza reglas condicionales creadas a partir de atributos de perfil, eventos contextuales y audiencias para adaptar los mensajes a perfiles de destino.

>[!ENDSHADEBOX]

>[!CONTEXTUALHELP]
>id="ajo_conditions_list"
>title="Condiciones"
>abstract="Las reglas condicionales permiten mostrar numerosas variantes de contenido en sus mensajes en función de atributos de perfil, eventos contextuales o públicos."

El contenido dinámico le permite adaptar el contenido de sus mensajes en función de las **reglas condicionales** que pueden estar formadas por atributos de perfil, eventos contextuales o públicos. Las reglas condicionales se crean mediante un generador de reglas visual dentro del editor de personalización, donde puede almacenarlas para su reutilización en los recorridos y campañas.

Las reglas condicionales se pueden aprovechar en el Designer de correo electrónico y en el editor de personalización para **crear contenido dinámico** que se adaptará a los perfiles objetivo en sus mensajes.

Esta página presenta el contenido dinámico en Journey Optimizer: cómo las reglas condicionales creadas a partir de atributos de perfil, eventos contextuales y audiencias permiten que los mensajes se adapten a los perfiles a los que se dirigen.

>[!TAB Glosario]

* **Contenido dinámico**: contenido de mensaje que se adapta en función de reglas condicionales; se muestra contenido diferente a perfiles diferentes en función de si se cumplen las condiciones definidas. *(específico del producto)*
* **Regla condicional**: conjunto de criterios (creados a partir de atributos de perfil, eventos contextuales o audiencias) que determina qué variante de contenido recibe un perfil. *(específico del producto)*
* **Generador de reglas visuales**: el lienzo interactivo del editor de personalización usado para crear y almacenar reglas condicionales. *(específico del producto)*

>[!TAB Terminología]

* **Nombre canónico:** contenido dinámico — variantes: contenido condicional, contenido personalizado
* **Sinónimos:** &quot;reglas condicionales&quot; = &quot;condiciones&quot; (como se etiquetan en la interfaz de usuario)
* **No confunda:** el contenido dinámico (contenido que se adapta por perfil en función de reglas condicionales) ≠ contenido estático (el mismo contenido mostrado a todos los destinatarios independientemente de las condiciones)

>[!TAB Preguntas más frecuentes]

**Q: ¿Qué tipos de criterios pueden utilizar las reglas condicionales?**

Las reglas condicionales se pueden crear a partir de atributos de perfil, eventos contextuales y pertenencia a audiencias.

**Q: ¿Dónde se pueden usar reglas condicionales para crear contenido dinámico?**

En el Designer de correo electrónico (para crear variantes de componentes) y en el editor de personalización (para ajustar bloques de expresiones con condiciones). Ambos puntos de entrada se tratan en páginas vinculadas.

**Q: ¿Es posible reutilizar las reglas condicionales en recorridos y campañas?**

Sí. Las reglas condicionales se pueden almacenar para su reutilización en recorridos y campañas.
